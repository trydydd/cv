## Design Context

### Users
Primary: hiring managers, recruiters, interviewers, professional contacts — time-constrained and evaluative, should land on the best resume immediately. Secondary: technical reviewers and curious peers who want to inspect the multi-model resume experiment, compare variants, and understand how outputs were produced.

### Brand Personality
Precise, transparent, ambitious. A polished personal portfolio crossed with a reproducible systems lab: confident enough for non-technical viewers, explicit enough for technical reviewers, restrained enough that the resume remains the hero. Physical scene reference: a bright review room with one excellent printed resume on the table and a precise lab notebook beside it.

### Aesthetic Direction
Light theme, OKLCH tokens, mostly pure-white surface so resume content stays crisp, with a deep rose/plum primary (`oklch(0.5 0.2 340)`) as a controlled accent — not a saturated campaign page. Typography: Atkinson Hyperlegible (sans) for interface/resume chrome, optional Source Serif 4 for the largest landing headline only, JetBrains Mono reserved for metadata/hashes/model IDs (never used as the whole brand voice). Anti-references: generic beige AI portfolio template, dense dashboards/developer-tool chrome on the general-viewer path, uncurated file-dump feel for the technical gallery.

### Design Principles
1. Route by intent, then disappear: `Everyone` gets the gold resume with minimum friction; `Tech` gets the lab.
2. Trust through provenance: model, prompt, input, and output metadata visible where it helps decision-making.
3. Configuration over code churn: site behavior follows YAML registries.
4. Resume first, experiment second: technical sophistication supports credibility, never distracts from the candidate.
5. Target WCAG 2.2 AA; do not rely on color alone for tier/status; respect `prefers-reduced-motion`.
