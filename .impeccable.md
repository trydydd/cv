## Design Context

### Users
Primary: hiring managers, recruiters, interviewers, professional contacts — time-constrained and evaluative, should land on the best resume immediately. Secondary: technical reviewers and curious peers who want to inspect the multi-model resume experiment, compare variants, and understand how outputs were produced.

### Brand Personality
Precise, transparent, ambitious. A polished personal portfolio crossed with a reproducible systems lab: confident enough for non-technical viewers, explicit enough for technical reviewers, restrained enough that the resume remains the hero. The letterhead belongs to the candidate — the brand mark is "Willard Hucks"; the experiment is a labeled section, not the identity. Physical scene reference: a bright review room with one excellent printed resume on the table and a precise lab notebook beside it.

### Aesthetic Direction
Print/editorial: paper surfaces, hairline rules, and typographic hierarchy instead of cards, pills, gradients, or floating shadows. Light theme, OKLCH tokens: plum-tinted paper background (`oklch(0.972 0.007 340)`), sheet-white document surfaces, deep plum primary (`oklch(0.42 0.165 340)`) as a controlled accent. Typography: Besley (Clarendon serif, self-hosted via Fontsource) for display/headings and the brand mark, Atkinson Hyperlegible for interface and body text, JetBrains Mono reserved strictly for provenance data (kickers, tags, hashes, model IDs). Landing page is a title page with a table-of-contents; the gallery is a specimen ledger of hairline-ruled rows; the resume renders as a printed sheet on the table. Anti-references: generic beige AI portfolio template, pill buttons and badge chips, identical card grids, gradient washes, dense dashboards/developer-tool chrome on the general-viewer path.

### Design Principles
1. Route by intent, then disappear: `Resume` gets the gold resume with minimum friction; `Resume Gallery` gets the lab.
2. Trust through provenance: model, prompt, input, and output metadata visible where it helps decision-making.
3. Configuration over code churn: site behavior follows YAML registries.
4. Resume first, experiment second: technical sophistication supports credibility, never distracts from the candidate.
5. Target WCAG 2.2 AA; do not rely on color alone for tier/status; respect `prefers-reduced-motion`.
